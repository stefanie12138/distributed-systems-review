分布式系统 · 考前总结

一、分布式系统核心概念

1.1 分布式系统定义（必考）

多个网络互联、独立自治的计算节点，基于消息传递协作完成共同目标。

五要素 关键点
多个计算节点 进程/线程/虚拟机均可
网络互联 不限物理拓扑
独立自治 独立CPU、独立时钟、独立失效
消息传递 非共享内存（Share Nothing）
共同目标 单节点无法独立完成

必记辨析：

  • 多CPU单主板计算机 不是 分布式系统（共享内存SMP，非消息传递）
  • 同一物理机 可以 包含多个分布式节点（进程/虚拟机 + loopback 通信）

1.2 共享内存 vs 消息传递

共享内存 消息传递
通信 隐式（读写共享变量） 显式（send/recv）
耦合 紧耦合 松耦合
容错 差（单点崩溃全局影响） 好（独立失效）
扩展 受限于总线带宽 理论无限水平扩展

二、架构模式与负载均衡

2.1 五种架构模式

模式 特点 典型应用
Client-Server 客户端轻、服务端重 Web 服务
Master-Slave 主节点调度，从节点执行 GFS、MapReduce
Peer-to-Peer 无中心，节点平等 比特币、BitTorrent
总线模式 通过共享总线通信 消息队列
混合模式 组合多种模式 复杂系统

必记：GFS/MapReduce → Master-Slave；比特币 → P2P

2.2 负载均衡策略

策略 原理 适用场景
轮询 依次分发 后端能力均等
加权轮询 按权重比例 后端能力不均
最少连接数 发给连接最少者 长连接
IP哈希 客户端IP取哈希 后端有缓存时首选
最短响应时间 发给响应最快者 动态负载

必记：后端有缓存 → IP哈希（同客户端始终落同后端，缓存命中率高）

2.3 服务器三层架构

表现层（Web服务器）→ 业务逻辑层（应用服务器）→ 数据层（数据库）

好处：各层独立扩展、解耦、可替换

三、通信技术

3.1 RPC

核心思想：让远程函数调用像本地调用一样简单。

流程：Client → Client Stub → 网络 → Server Skeleton → Server

  • Stub：客户端代理，序列化参数
  • Skeleton：服务端代理，反序列化并调用实际函数
  • IDL：接口定义语言，支持跨语言调用
  • RPC中间件作用：屏蔽网络细节、序列化/反序列化、跨语言

3.2 消息中间件（MOM）

模式 特点
消息队列 (Queue) 点对点，一个消息一个消费者，FIFO
发布/订阅 (Topic) 一对多，一个消息多个订阅者

消费者接收消息三种方式：同步接收（阻塞等待）、异步接收（回调）、轮询接收

MOM优点：解耦、削峰填谷、异步、可靠投递

秒杀/双十一：请求入队列→后端按能力消费→保护后端不冲垮

3.3 并发服务技术

技术 适用场景
多线程 通用，连接数有限
线程池 复用线程
事件驱动 (Reactor) 大量短连接
Proactor 异步I/O
协程 高并发、轻量级

四、分布式模型与时钟（重难点）

4.1 四种网络链路模型（从弱到强）

任意链路（有主动攻击）→ 一般损失链路（无主动攻击，丢包/重复/乱序）→ 可靠链路（最终必达，可能乱序）→ 可靠FIFO链路（保序）

实现手段：纠错码+HASH+MAC+加密 → 重传+ACK+序列号去重 → 序列号排序

4.2 两种节点故障模式

Fail-Stop Byzantine
行为 安静停止 任意动作（发错误消息）
攻击者 无恶意 有恶意（黑客控制）
容错要求 $n > f$ 或 $n > 2f$ $n > 3f$
应用 常规分布式系统 航天、金融、区块链

4.3 物理时钟同步 — Cristian算法（NTP基础）

  1. 客户端 $T_1$ 发请求
  2. 服务器 $T_2$ 收到，$T_3$ 回复（带 $T_2, T_3$）
  3. 客户端 $T_4$ 收到

$$\delta = (T_4 - T_1) - (T_3 - T_2)$$

关键假设：$\delta_{req} \approx \delta_{resp}$

$$T_{client} \leftarrow T_3 + \frac{\delta}{2} \quad \text{误差：} \pm \frac{\delta}{2}$$

4.4 Lamport 逻辑时钟（标量时钟）

Happen-Before ($\rightarrow$)：同进程先后 → 消息发送先于接收 → 传递性

规则：

  • 内部/发送事件：$C_i \leftarrow C_i + 1$
  • 接收消息（带 $C_{msg}$）：$C_j \leftarrow 1 + \max(C_j, C_{msg})$

核心性质：

$$a \rightarrow b \Rightarrow C(a) < C(b) \quad \text{✓ 因果保持}$$

$$C(a) < C(b) \nRightarrow a \rightarrow b \quad \text{✗ 反向不成立！}$$

4.5 全序逻辑时钟

二元组排序：$\langle C, ID \rangle$

$$\begin{aligned} L(a) < L(b) \iff & C(a) < C(b) \;\text{OR}\; \\ & \big(C(a)=C(b) \;\text{AND}\; ID(a)

4.6 向量逻辑时钟

每个节点维护 n 维向量：$\mathbf{T}_i = \langle t_1, \ldots, t_n \rangle$

  • 内部事件：$\mathbf{T}_i[i] \leftarrow \mathbf{T}_i[i] + 1$
  • 接收消息：逐维取 max，再自增

充要条件：$a \rightarrow b \iff V(a) < V(b)$（能双向判断因果与并发）

4.7 Paxos / Raft 共识

最大容错：$f_{max} = \lfloor (n-1)/2 \rfloor$（需多数派存活）

Raft角色：Leader / Follower / Candidate

4.8 一致性模型（从强到弱）

强一致性 最强 写后立即可读最新，全局顺序
顺序一致性 强 按程序顺序，不要求实时
因果一致性 中 因果保序，并发任意序
最终一致性 弱 停止写入后最终一致

判断：主节点写→等所有副本确认→返回成功 → 满足强一致性；读可从任意副本 → 可能过期读

五、MapReduce 与大数据生态

5.1 MR 计算模型

三阶段：Split → Map（第一阶段并行）→ Shuffle（二次分区）→ Reduce（第二阶段并行）→ Output

框架作用（7项）：自动分片、任务分发、负载感知、数据本地性、混洗合并、容错、可扩展

5.2 HDFS

组件 功能
NameNode 管理元数据（目录结构、文件→块映射、块→DataNode映射）
DataNode 存储实际数据块（默认128MB）

读取流程：客户端→NameNode（获取块位置）→DataNode（读数据）

可靠性：多副本（默认3）、机架感知、心跳检测、安全模式

小文件问题：每条元数据~150B，大量小文件→NameNode内存爆炸

5.3 Spark vs MapReduce

MapReduce Spark
中间结果 写磁盘 内存RDD（快10~100倍）
计算模型 Map+Reduce Transformation+Action
容错 任务重试 RDD血统(lineage)重算

5.4 一致性哈希

空间：$[0, 2^{32}-1]$ 环形

定位：$H(\text{key})$ 沿环顺时针第一个服务器节点

增加节点：仅移动新节点与前驱之间的数据

计算题步骤：①服务器哈希排序 ②数据哈希定位 ③确定归属 ④计算迁移量

5.5 MR 编程模板

TopN：Map输出TopN → Combine局部合并 → Reduce全局TopN

微博大V：Map输出 <被关注者, 1> → Reduce求和粉丝数 → 过滤>50万

六、MPI 并行编程

6.1 核心概念

  • MPI = 函数库规范（非语言），常见实现：Open MPI、MPICH
  • SPMD 模型（单程序多数据），通过 rank 区分进程

6.2 核心 API

MPI_Init/Finalize（环境）、Comm_rank/size（标识）、Send/Recv（阻塞点对点）、Isend/Irecv（非阻塞）

6.3 阻塞 vs 非阻塞

阻塞 非阻塞
返回时机 操作完成 立即返回request
缓冲区 返回后可复用 需Wait()确认
计算通信重叠 不能 可以

6.4 六大集合通信

操作 函数 数据流向
广播 bcast 根→所有进程
散播 scatter 数组→各进程各取一元素
收集 gather 各进程→根进程汇总
归约 reduce 各进程→根进程归约(SUM/MAX/...)
全收集 allgather gather+广播
全归约 allreduce reduce+广播

九、易错点

  1. Lamport逻辑时钟：$C(a) < C(b) \nRightarrow a \rightarrow b$（反向不成立！）
  2. 向量时钟：$V(a) < V(b) \iff a \rightarrow b$（充要条件）
  3. Fail-Stop vs Byzantine：容错公式不同（$n>f$ vs $n>3f$）
  4. 一致性哈希：增节点只移动相邻区间数据，非全部重新分布
  5. HDFS小文件：瓶颈在NameNode内存，非DataNode磁盘
  6. MapReduce Shuffle：按Key二次分区，非随机打乱
  7. RPC vs MOM：RPC同步请求-响应；MOM异步消息
  8. MPI阻塞：Send完成后缓冲区可复用；非阻塞需Wait确认
